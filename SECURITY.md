# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | ✅ Full support    |
| < 1.0   | ❌ Not supported   |

## Reporting a Vulnerability

如果你发现安全漏洞，请通过以下方式报告：

### 联系方式
- **电子邮件**: security@codepop.dev
- **PGP 密钥**: 可应要求提供

### 报告内容
请在报告中包含：
- 漏洞描述
- 复现步骤
- 受影响的版本
- 潜在影响评估

### 响应时间
- 我们会在 **24 小时内**确认收到报告
- 在 **7 天内**提供初步评估和修复计划
- 在 **30 天内**发布安全修复

### 披露政策
我们遵循协调披露原则：
1. 漏洞报告保密处理
2. 修复发布后公开披露
3. 会在致谢名单中提及贡献者

## 安全最佳实践

### 开发人员
- 定期更新依赖
- 使用安全的编码实践
- 避免硬编码敏感信息
- 所有输入必须验证和清理

### 用户
- 保持软件更新到最新版本
- 使用强密码
- 仅在安全网络中访问服务
- 定期审查应用权限
